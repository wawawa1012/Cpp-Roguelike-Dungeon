# C++ Roguelike 地牢生成器 (Hades 2 风格)

基于现代 C++ 标准实现的 2D 网格随机房间生成算法，逐步演进的商业级架构重构。

## Day 1: 核心机制与算法基石
1. **高质量随机引擎**: 弃用 `rand()`，改用 `<random>` 库（梅森旋转算法 `std::mt19937`），确保坐标分布绝对均匀。
2. **AABB 碰撞检测**: 采用反向排除法，带有 1 格接壤缓冲带，大幅简化重叠判断逻辑。
3. **拒绝采样与熔断机制**: 引入最大尝试次数 (`MAX_ATTEMPTS`) 熔断计数器，解决空间极度拥挤时的死循环 Bug。

## Day 2: 最小生成树与图论网络
1. **数学建模**: 定义 `Edge` 结构体，计算曼哈顿距离作为边权。
2. **并查集 (Union-Find)**: 独立实现带有“路径压缩”优化的并查集算法，完美拦截回路死锁。
3. **Kruskal 算法连通**: 结合抛硬币随机机制挖掘 L 型走廊，完成全图连通。

## Day 3: 商业级工程化重构
1. **打破面条代码 (Spaghetti Code)**: 物理拆分文件，实现高内聚低耦合。
2. **内存布局控制**: 剔除多余库依赖，严禁头文件污染命名空间，确保类内存布局完整性。

## Day 4: 游戏引擎化、交互系统与实体降临
1. **游戏主循环 (Game Loop)**: 引入 `play()` 函数，构建 Render-Input-Update 标准引擎循环机制。
2. **无阻塞 I/O & 双缓冲渲染**: 使用 `<conio.h>` 的 `_getch()` 实现瞬间键盘响应，调用 Windows API 隐藏光标并复位 `(0,0)` 覆盖旧画面，彻底消灭闪屏。
3. **数据驱动的房间语义化**: 引入 `enum class RoomType`，基于中心点曼哈顿距离划分出生房 (S)、Boss 房 (B)、宝藏房 (T) 和普通怪物房 (.)。
4. **实体组件建模与拾取机制**: 构建 `Enemy` 与 `Player` 结构体。实现玩家移动碰撞阻挡，以及触发宝藏 (T) 的状态修改与销毁逻辑。
5. **区域限制刷怪 (DRY 原则)**: 基于选定房间矩形区域动态调整随机数分布范围，结合 `for` 循环实现批量刷怪，彻底避免硬编码。

## Day 5: 实体 AI、撞击战斗与生死轮回
1. **意图与执行分离**: 重构 `switch-case` 逻辑，采用“先预算目标坐标，后统一处理逻辑”的架构，极大提升了代码的 DRY (Don't Repeat Yourself) 纯净度。
2. **撞击战斗引擎 (Bump Combat)**: 实现了实体的伤害结算。玩家通过向怪物 (E) 方向移动触发攻击，包含了死亡判定与 `std::vector` 容器的安全删除逻辑。
3. **贪心寻路 AI**:
    * 基于曼哈顿距离实现“视野仇恨”机制。
    * 编写了二轴贪心寻路算法，并成功修复了导致怪物乱跳的“零值陷阱”逻辑漏洞。
4. **属性面板 (HUD)**: 动态显示 HP (血量)、ATK (攻击力) 与宝藏数，实时反馈战斗状态。
5. **轮回机制 (Permadeath)**:
    * 在 `play()` 中埋入死亡跳出逻辑。
    * 利用 C++ 对象的生命周期 (Object Lifecycle)，在 `main()` 函数中构建重生闭环，确保每一局游戏都是全新的内存空间与随机体验。
